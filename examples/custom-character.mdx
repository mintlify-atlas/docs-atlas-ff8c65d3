Define rich, detailed characters that shape how your agents think, communicate, and interact. This example shows how to create agents with distinct personalities, knowledge bases, and behavioral patterns.

## Overview

Characters are the foundation of your agent's identity. They define personality traits, communication style, knowledge domains, and behavioral guidelines.

**What you'll learn:**
- Define character traits and personality
- Add knowledge and topics
- Set communication style
- Configure behavioral patterns
- Load characters from JSON files

---

## Quick Start

<Steps>
  <Step title="Create Character File">
    Create `character.json` with your character definition
  </Step>
  
  <Step title="Load Character">
    ```typescript
    import { loadCharacter } from "@elizaos/core";
    const character = await loadCharacter("./character.json");
    ```
  </Step>
  
  <Step title="Create Runtime">
    ```typescript
    const runtime = new AgentRuntime({ character, plugins });
    await runtime.initialize();
    ```
  </Step>
</Steps>

## Using the Character

```typescript agent.ts
import { AgentRuntime, loadCharacter } from "@elizaos/core";
import { openaiPlugin } from "@elizaos/plugin-openai";
import { plugin as sqlPlugin } from "@elizaos/plugin-sql";

// Load character from file
const character = await loadCharacter("./character.json");

console.log(`Creating agent: ${character.name}`);
console.log(`Personality: ${character.bio}`);

// Create runtime with character
const runtime = new AgentRuntime({
  character,
  plugins: [sqlPlugin, openaiPlugin],
});

await runtime.initialize();

// The agent now embodies the character's personality
const response = await runtime.useModel("TEXT_LARGE", {
  prompt: "Tell me about renewable energy",
});

console.log(response);
// Expected: Enthusiastic, technical response about renewable energy
// in Tesla's characteristic style
```

## Character Components

### Basic Information

```json
{
  "name": "Character display name",
  "username": "unique_identifier",
  "bio": "Short description visible to users"
}
```

### System Prompt

The `system` field defines core behavioral instructions:

```json
{
  "system": "You are [name], a [role] who [characteristics]. You [behavior patterns]. When responding, you [communication style]."
}
```

<Tip>
The system prompt is the most important field for shaping behavior. Be specific and detailed.
</Tip>

### Traits

Define personality characteristics:

```json
{
  "traits": [
    "curious",      // Asks questions
    "analytical",   // Breaks down complex topics
    "empathetic",   // Shows understanding
    "humorous",     // Uses appropriate humor
    "patient"       // Takes time to explain
  ]
}
```

### Topics

Specify areas of expertise:

```json
{
  "topics": [
    "machine learning",
    "data science",
    "python programming",
    "statistics"
  ]
}
```

### Style Guide

Control communication style:

```json
{
  "style": {
    "tone": "professional yet friendly",
    "vocabulary": "technical terms with explanations",
    "engagement": "asks clarifying questions",
    "formality": "casual in DMs, formal in public"
  }
}
```

### Message Examples

Provide conversation samples for few-shot learning:

```json
{
  "messageExamples": [
    [
      { "user": "user", "content": { "text": "Question" } },
      { "user": "Agent", "content": { "text": "Response in character" } }
    ]
  ]
}
```

<Tip>
Include 5-10 message examples showing your character's communication style and knowledge.
</Tip>

## Advanced Features

### Knowledge Base

Add specific domain knowledge:

```json
{
  "knowledge": [
    "Detailed fact or concept your agent should know",
    "Expertise in specific area with nuances",
    "Understanding of common misconceptions in the field"
  ]
}
```

### Lore (Background Story)

```json
{
  "lore": [
    "Created by a team of researchers at...",
    "Specialized training on scientific papers from...",
    "Designed to make complex topics accessible"
  ]
}
```

### Post Examples (for Social Media)

```json
{
  "postExamples": [
    "Tweet-style message in character",
    "Share interesting fact with personality",
    "Thought-provoking question to followers"
  ]
}
```

## Loading Characters

### From JSON File

```typescript
import { loadCharacter } from "@elizaos/core";

const character = await loadCharacter("./character.json");
```

### From Object

```typescript
import { validateCharacter } from "@elizaos/core";

const character = {
  name: "MyAgent",
  bio: "A helpful assistant",
  system: "You are helpful and concise.",
};

// Validate before using
const validated = validateCharacter(character);
```

### From Multiple Sources

```typescript
import { mergeCharacters } from "@elizaos/core";

const base = await loadCharacter("./base.json");
const custom = await loadCharacter("./custom.json");

// Merge with custom overriding base
const character = mergeCharacters(base, custom);
```

## Best Practices

<Tip>
**Be Specific**: Vague traits like "smart" are less effective than specific behaviors like "breaks down complex topics into simple analogies."
</Tip>

<Tip>
**Show, Don't Tell**: Use message examples to demonstrate the character's style rather than just describing it.
</Tip>

<Tip>
**Consistent Voice**: Ensure all fields (bio, system, examples) reflect the same personality and tone.
</Tip>

<Tip>
**Test Thoroughly**: Chat with your character extensively to ensure it behaves as expected across different scenarios.
</Tip>
